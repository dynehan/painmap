# 2주차 1일차(월) — 오늘 계획

오전엔 이번 주 계획을 세우고 실제 이슈로 옮기고, 오후엔 계획 수립 Agent를 만들어 오늘 세운 계획을 점검한다.

## 1. 이번 주(2주차) 목표 재확인

`docs/빵지도_Task_백로그.md`에 이미 정리해둔 2주차 목표: **백엔드 기반 + 데이터 수집 착수**

- 저장소 세팅 (client/server 모노레포)
- 백엔드 API 서버 기본 구조 (Express)
- DB 스키마 설계 (빵집 테이블, 회원 테이블)
- 카카오맵 JS SDK 연동
- 빵집 데이터 수집 착수, 하루 큐레이션 가능량 체크
- 회원가입/로그인 API (bcrypt + JWT)

금요일까지 목표: **카카오맵에 실제 빵집 데이터가 마커로 뜨고, 로그인/회원가입 API가 동작하는 상태.**

## 3. 어디에 저장할까 — 결정

캠프 지침상 "대시보드(GitHub Issues·Projects, Notion, Google Sheet 중 하나)"로 관리해야 함. 지난주에 만든 `빵지도_Task_백로그.md`는 큰 그림 로드맵 문서로 유지하고, **실제 작업 단위 추적은 오늘부터 GitHub Projects(칸반보드) + Issues로 이전**한다.

- 칸반 컬럼: `작업 중` / `다음 작업` / `백로그`
- 각 이슈 제목 형식: `[FE/BE/DB] 작업명`
- 이슈 본문에 완료 기준 1~2줄 + 우선순위 라벨(P0/P1/P2) 부여
- README에 프로젝트 보드 링크 추가

**참고 — 열어둘 논의**: 캠프 예시 칸반에는 DB로 Supabase가 쓰여 있는데, `CLAUDE.md`에는 개발 편의를 이유로 better-sqlite3로 정해뒀음. 오늘 그룹세션에서 Supabase(관리형, 실시간/인증 기능 내장) vs SQLite(설치 없이 바로 시작) 중 뭘 쓸지 한 번 더 확인하고 확정하는 게 좋을 것 같음.

## 5. 계획 수립 Agent 만들기 (오후)

**역할**: 기능 요구사항을 입력하면, 하루 안에 끝낼 크기의 작업 단위로 쪼개고 우선순위·완료기준·FE/BE/DB 태그를 붙여 목록으로 돌려주는 Agent.

- **입력**: 자연어로 된 기능 요구사항 (예: "동선 계산 기능 만들어줘")
- **출력**: 작업명 / 완료 기준 / 우선순위(P0~P2) / 예상 태그(FE/BE/DB) 목록
- **사용 예시**: `"동선 계산 기능을 만들기 위해서 'task-planner' agent를 사용해서 계획을 나눠줘"`
- 오늘 세운 위 표(4번)를 이 Agent에 다시 넣어 결과를 비교해보고, 누락된 작업이나 잘못된 우선순위가 있는지 점검한다.

1. 2주차 목표는 백엔드 기반 + 데이터 수집 착수, 금요일까지 지도에 실데이터 + 로그인 API 완성이 목표.
2. 작업 관리는 오늘부터 GitHub Projects(칸반) + Issues로 옮기고, README에 링크.
