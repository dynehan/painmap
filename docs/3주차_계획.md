# 3주차 계획 — 수직슬라이스 완성

## 지금 상태 요약

2주차까지: 화면(지도/리스트/경로결과/챗봇/마이페이지/로그인·회원가입) UX 전부 완성, 네이버지도 실연동, 내 위치+출발점 고정 동선 계산까지 프론트에서 동작. 백엔드는 Express+Supabase(Postgres) 기반 잡혔고 회원가입/로그인은 bcrypt+JWT로 실제 API 연동 검증 완료.

**핵심 한계**: 빵집 데이터가 전부 mock(가짜 6곳)이라 지도/리스트/동선 계산이 전부 프론트 안에서만 도는 임시 로직. 서버·DB랑 실제로 연결된 화면-서버-DB 수직슬라이스가 아직 없음.

오늘 캠프 미션도 정확히 이 지점을 짚음: "수직슬라이스 안 됐으면 이번주 1순위는 기능 강화가 아니라 수직슬라이스 완성." → **3주차 전체 목표를 수직슬라이스 완성으로 확정.**

참고로 빵집 데이터는 지난주에 이미 착수해뒀어요 — 2025 대전빵축제 참가업체 102곳 명단 확보, 그중 10곳은 주소·영업시간까지 검색으로 채워둔 상태(성심당·하레하레·관저당·손수베이커리·콜마르브레드·마들렌과자점·뮤제·캘리포니아베이커리·굿베이커리·정인구팥빵). 이 부분은 클로드코드 쪽엔 아직 공유 안 됐으니, 이번주 데이터 작업은 이걸 이어받아서 하면 됩니다.

## 이번주 처리 순서 (백로그 그대로, 우선순위 재확인)

1. 빵집 실데이터 DB 반영 — **가장 먼저**, 이게 풀려야 나머지가 다 풀림
2. 빵집 리스트 API 실연동 (mock 배열 → 서버 응답으로 교체)
3. 동선 계산 서버 이전 (완전탐색/휴리스틱을 서버 API로)
4. 로그인 상태 유지 + 마이페이지 실데이터
5. 부가 기능(마감임박, 찜, 코스저장 등) — 여유될 때만, 이번주 우선순위 아님
6. 막판 다듬기/데모 리허설 — 4주차 몫, 이번주 범위 아님
