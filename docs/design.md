# Premium Soft Dashboard Design Skill

## Goal

Create interfaces with the same design language as modern premium SaaS dashboards seen on Dribbble.

The UI should feel calm, spacious, elegant, premium, and effortless rather than colorful or overly decorative.

Think:
- Apple
- Linear
- Arc Browser
- Notion
- Modern AI Dashboard

This is a design language, **NOT a layout to copy.**

---

# Core Design Philosophy

Minimal elements.

Maximum whitespace.

Every component should feel intentionally placed.

Information density should remain moderate.

Avoid visual noise.

The interface should feel expensive rather than impressive.

# Layout

Prefer large card-based layouts.

Large margins around every section.

Consistent spacing throughout the page.

Use visual breathing room aggressively.

Typical spacing scale

- 8
- 12
- 16
- 24
- 32
- 40
- 48

Cards should align perfectly.

No cramped UI.

---

# Background

Primary background

Warm white

Examples

- #FAFAF8
- #FCFCFB
- #F8F8F6

Optional

Very soft radial gradients

Examples

- pale yellow
- pale green
- pale blue

Opacity should remain extremely subtle.

Never use aggressive gradients.

---

# Cards

Cards are the primary visual element.

Properties

- white
- large border radius
- almost flat
- extremely subtle shadow

Border Radius

24~32px

Border

1px

Very low contrast

Example

rgba(0,0,0,0.04)

Shadow

Very soft.

Almost invisible.

Never use heavy shadows.

---

# Typography

Use clean sans-serif fonts.

Recommended

- Inter
- Geist
- SF Pro
- Plus Jakarta Sans

Hierarchy

Large title

40~48px

Section title

24~32px

Card title

18~22px

Body

14~16px

Caption

12~13px

Font weight

400
500
600

Avoid bold unless necessary.

---

# Colors

Use very few colors.

Primary text

Almost black

#1B1B1B

Secondary text

Gray

#7B7B7B

Tertiary

#A8A8A8

Accent

Only one accent color throughout the interface.

Examples

Green
Blue
Orange

Never use many accent colors.

---

# Charts

Charts should feel like illustrations.

Avoid

- thick axes
- dark grids
- loud colors

Prefer

- thin lines
- pastel gradients
- soft fills
- minimal labels

Charts should enhance the interface,
not dominate it.

---

# Icons

Use

- Lucide
- Heroicons

Style

Outline

2px

Small

Minimal

Never use filled icons unless absolutely necessary.

---

# Buttons

Primary

Black

Rounded pill

White text

Secondary

White

Thin border

Rounded

Padding

Generous

Never make buttons look heavy.

---

# Inputs

Rounded

Soft border

No strong outlines

Very subtle focus states

Placeholder

Light gray

---

# Images

Images should

- feel editorial
- have generous corner radius
- blend naturally into cards

Avoid stock-photo feeling.

---

# Motion

Animation should be subtle.

Think Apple.

Duration

200~350ms

Use

- fade
- translateY
- scale 0.98 → 1
- opacity

Never use

- bounce
- elastic
- flashy transitions

Hover

Tiny lift

Tiny shadow increase

Nothing more.

---

Page

None

Cards

28px

Buttons

999px

Input

18px

Small tags

999px

Charts

---

# Shadows

Very restrained.

Examples

0 4px 16px rgba(0,0,0,.04)

or

0 12px 32px rgba(0,0,0,.05)

Avoid obvious shadows.

---

# Visual Hierarchy

Large whitespace first.

Typography second.

Cards third.

Color last.

Color should never be responsible for hierarchy.

Hierarchy comes from

- spacing
- size
- alignment

---

# Do

✓ Lots of whitespace

✓ Large rounded cards

✓ Thin typography

✓ Soft gradients

✓ Clean alignment

✓ Editorial composition

✓ Calm interactions

✓ Minimal colors

✓ Premium feeling

---

# Don't

✗ Glassmorphism

✗ Skeuomorphism

✗ Heavy neumorphism

✗ Strong shadows

✗ Neon colors

✗ Busy dashboards

✗ Dense tables

✗ Excessive borders

✗ Multiple accent colors

✗ Thick icons

✗ Loud animations

---

# Golden Rule

Every screen should look like it could appear on the front page of Dribbble as a premium AI SaaS product.

Whenever unsure, remove visual elements instead of adding them.

Less is almost always better.
