# 빵지도 프로토타입 — 추가 기능 스펙

대상 파일: `빵지도_prototype_v2.html` (기존 v2 리디자인 버전에 아래 기능들을 추가 반영)

기존 구조 참고:
- 데이터: `bakeries` 배열 (id, name, menu, x, y, phone, busy)
- 화면: `#screen-map`, `#screen-list`, `#screen-route`, `#screen-chat`, `#screen-mypage`
- 색상 변수: `--yellow`, `--yellow-soft`, `--ink`, `--bg` 등 기존 CSS 변수 재사용
- 마스코트: `<img src="character.png" class="mascot-icon">` (또는 SVG `#mascot` 심볼), 로고는 `logo.png`

아래 10개 기능을 전부 추가합니다.

---

## 1. 지도 위 말풍선 표시 — 마감임박 / 한산해요 (★ 우선 구현, 핵심 요구사항)

**중요**: 클릭/호버 없이 지도에서 바로 보여야 함. 마커 옆에 항상 떠 있는 말풍선(speech bubble) 형태.

- 각 빵집 데이터에 영업시간 필드 추가: `openHour`, `closeHour` (예: 9, 20 — 24시간제 숫자)
- 데모용 "현재 시각"은 하드코딩된 변수로 시뮬레이션 (예: `const demoNow = 19.5;` 처럼 조절 가능하게 해서, 실제 Date 대신 데모에서 자유롭게 상황을 재현할 수 있게 함)
- 조건별 말풍선:
  - **마감임박**: `closeHour - demoNow <= 1` (마감 1시간 이내) → 마커 위에 빨강/주황 계열 말풍선, 텍스트 "곧 마감"
  - **한산해요**: 현재 시각이 `busy`(몰리는 시간) 데이터 범위에 포함되지 않을 때 → 초록 계열 말풍선, 텍스트 "한산해요"
  - 두 조건이 동시에 해당하면 마감임박을 우선 표시
- 스타일: 마커 라벨(`.pin .label`) 위쪽에 작은 말풍선 뱃지 추가. 꼬리(tail)가 마커를 향하는 CSS 삼각형, 배경은 상태별 컬러(마감임박: `#E1596B` 계열 / 한산: `#3FA07D` 계열), 흰 텍스트, 8~9px 정도의 작은 폰트. 지도가 복잡해 보이지 않도록 해당 조건에 해당하는 마커에만 표시.
- 리스트 카드에도 동일한 뱃지를 이름 옆에 작게 표시 (지도와 리스트 상태 일치).

## 2. 찜하기(하트) 아이콘

- 지도 상세 패널과 리스트 카드 각각에 하트 아이콘 버튼 추가 (기존 "선택하기"와는 별개 — 동선 계산용 선택이 아니라 순수 찜/즐겨찾기).
- 클릭 시 채워진 하트로 토글, `wishlist` Set에 id 저장.
- 마이페이지의 "가고 싶은 곳" 목록과 연동 (회원가입 때 고른 목록 + 찜한 항목 합쳐서 표시).
- 라인 아이콘(outline) → 채워진 아이콘으로 상태 전환, 색상은 `--yellow-deep` 또는 코럴 계열.

## 3. 리스트 화면 필터 / 정렬

- 리스트 화면 상단에 필터 바 추가: 빵 종류 칩(크루아상/식빵/단팥빵 등 다중 선택), 가격대 슬라이더 또는 3단계 버튼(저/중/고), "지금 영업중만 보기" 토글.
- 정렬 드롭다운: 이름순 / 가까운 순(추후 위치 연동) / 찜 많은 순(더미 데이터 기준).
- 데이터에 `price`(1~3 등급), `category`(빵 종류 배열) 필드 추가 필요.
- 필터링은 순수 JS 배열 filter로 리스트 그리드 재렌더링.

## 4. 검색바 실제 동작

- 상단 검색창(`.search`)을 실제 `<input>`으로 교체.
- 입력값으로 `name`, `menu` 필드 매칭해 리스트/지도 마커를 실시간 필터링 (매칭 안 되는 마커는 흐리게 처리하거나 숨김).

## 5. 경로 결과 공유하기

- 경로 결과 화면(`#screen-route`)에 "공유하기" 버튼 추가.
- 클릭 시 방문 순서 텍스트("성심당 본점 → 앤드유 → 하레하레")를 클립보드에 복사하고 토스트 메시지 "링크가 복사되었습니다" 표시 (프로토타입 단계라 실제 카카오톡 SDK 연동은 생략, `navigator.clipboard.writeText` 사용).

## 6. 코스 저장 / 이름 붙이기

- 경로 결과 화면에서 "이 코스 저장하기" 버튼 → 모달로 코스 이름 입력(예: "대학가 빵투어").
- 저장된 코스는 `savedCourses` 배열(메모리 상태)에 { name, order(가게 id 배열) } 형태로 저장.
- 마이페이지에 "저장한 코스" 섹션 추가, 목록 클릭 시 해당 동선으로 다시 경로 결과 화면 진입.

## 7. 리뷰 / 별점

- 빵집 상세 패널에 별점(1~5, 더미 평균값 예: 4.3) 표시.
- 한줄 리뷰 목록(더미 2~3개)을 상세 패널 하단에 카드로 추가.
- 데이터에 `rating`, `reviews`(문자열 배열) 필드 추가.

## 8. 방문 스탬프 / 뱃지 (마스코트 연계)

- 마이페이지에 "스탬프 투어" 섹션 추가: 방문한 곳 수에 따라 마스코트 캐릭터 아이콘이 순서대로 채워지는 스탬프 그리드 (예: 3곳 방문 시 스탬프 3개 활성화).
- 특정 개수 도달 시 뱃지 문구 표시 (예: "5곳 방문 — 빵지도 초보 탐험가", "10곳 — 빵지도 마스터") — 마스코트 아이콘과 함께.

## 9. 내 위치 기준 정렬

- 리스트 화면 정렬 옵션에 "내 주변 가까운 순" 추가.
- 실제 GPS 대신 데모용 고정 좌표(예: 지도 중앙값)를 "내 위치"로 가정하고 거리순 정렬 (기존 `dist()` 함수 재사용).

## 10. 다크모드 토글 (선택 사항, 여유 있으면)

- 상단바에 라이트/다크 토글 버튼 추가.
- CSS 변수(`--bg`, `--surface`, `--ink` 등)를 다크 세트로 스왑하는 `data-theme="dark"` 클래스 방식.

---

## 데이터 모델 변경 요약

`bakeries` 배열 각 항목에 아래 필드 추가 필요:

```js
{
  id, name, menu, x, y, phone, busy,   // 기존
  openHour, closeHour,                  // 1번 마감임박용
  price,                                // 3번 가격대 (1~3)
  category: ['크루아상', ...],           // 3번 빵 종류
  rating,                               // 7번 별점
  reviews: ['...', '...']               // 7번 리뷰
}
```

## 우선순위 제안

1차(바로): 1번(마감/한산 말풍선), 2번(찜하기), 4번(검색) — 기존 데이터로 바로 구현 가능
2차: 3번(필터/정렬), 5번(공유), 9번(내 주변)
3차: 6번(코스저장), 7번(리뷰), 8번(스탬프), 10번(다크모드) — 상태 저장/UI가 더 필요한 것들
