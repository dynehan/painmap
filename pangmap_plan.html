문제는 빵이 아니라
동선입니다.

대전 빵투어는 갈 곳이 많을수록 방문 순서, 이동 시간, 마감시간, 품절 가능성을 함께 고려해야 합니다.

Problem

방문 순서 고민

여러 빵집을 하루에 돌고 싶지만 어디부터 가야 효율적인지 사용자가 직접 비교해야 합니다.

Problem

이동 시간 낭비

지도 앱은 개별 길찾기는 잘하지만, 여러 장소를 한 번에 고려한 최적 순서 제안은 약합니다.

Problem

마감·품절 변수

마감시간에 맞춰 도착해도 인기 메뉴가 품절된 경우가 있어 빵투어 만족도가 떨어질 수 있습니다.

핵심은
AI 동선 설계.

단순한 빵집 리스트가 아니라, 사용자가 선택한 장소들을 실제로 어떻게 돌아야 하는지 알려주는 서비스입니다.

핵심 기능

선택한 빵집의 상위 3개 경로 자동 추천

사용자가 지도나 리스트에서 빵집을 고르면, 가장 효율적인 방문 순서 3개를 계산해 보여줍니다. 1순위 경로는 지도 색상과 리스트 색상을 일치시켜 직관적으로 안내합니다.

1위 성심당 → 앤드유 → 하레하레추천
2위 앤드유 → 성심당 → 하레하레대안
3위 하레하레 → 앤드유 → 성심당대안
Route

이동수단별 시간 비교

도보, 자동차, 버스별 예상 소요시간을 제공해 사용자가 상황에 맞는 이동 방식을 고를 수 있게 합니다.

Account

로그인 / 회원가입

아이디·비밀번호 기반 계정과 빵 취향, 가본 곳, 가고 싶은 곳 데이터를 저장합니다.

Curation

대화형 빵집 큐레이터

“달달한 거 먹고 싶어”처럼 자연어로 요청하면 보유 데이터 중 적합한 빵집을 추천합니다. 단, 부가 기능으로 후순위 구현합니다.

MVP

15~20곳 데이터 집중

4주차 데모 기준으로 대전 전역 15~20곳의 베이커리 데이터를 큐레이션해 완성도를 높입니다.

데이터는
작지만 진하게.

공공데이터 기반 정보에 직접 조사한 큐레이션 정보를 더해, 단순 위치 정보 이상의 빵집 선택 기준을 제공합니다.

User Scenario

다인이의
주말 빵투어

앱을 켜고, 지도와 리스트를 오가며 빵집을 고르고, 선택한 장소를 바탕으로 가장 효율적인 방문 순서를 추천받습니다.

1
지도에서 탐색홈 화면에서 대전 베이커리 마커를 확인합니다.
2
리스트에서 비교성심당 본점, 하레하레, 앤드유를 체크합니다.
3
선택함 확인고른 빵집 목록을 확인하고 동선 계산을 누릅니다.
4
추천 경로로 출발가장 효율적인 순서와 이동수단별 시간을 보고 빵투어를 시작합니다.

화면 구성은
명확하게.

데스크톱 브라우저 기준으로 상단 검색바와 하단 메뉴바를 기본 구조로 설계합니다.

기술 구조는
직접 구축 중심.

프론트엔드, 백엔드, DB, 지도 API, 동선 계산 로직을 분리해 MVP 이후 확장 가능하게 설계합니다.

Client

React + Vite 웹 앱
지도 · 리스트 · 경로 결과 · 챗봇 · 마이페이지
↓ HTTP 요청

Server

Node.js API 서버
DB 조회 / 회원 인증 / 동선 계산
완전탐색 + 휴리스틱 알고리즘

External

Naver Maps JavaScript API
길찾기·대중교통 API 2단계 도입
LLM API 챗봇 부가 기능

리스크는
범위 조절로 대응.

4주 안에 데모 가능한 수준을 확보하기 위해 핵심 기능과 부가 기능을 명확히 분리합니다.

백엔드 구축 시간 증가데이터 목표치를 15~20곳으로 조정해 구현 시간을 확보합니다.
인증 기능 범위 확대소셜 로그인, 비밀번호 찾기는 제외하고 아이디·비밀번호 기반으로 한정합니다.
대중교통 API 난이도1차는 거리 기반 근사치로 구현하고, 실제 API는 시간이 남을 때 교체합니다.
경로 계산 복잡도소규모는 완전탐색, 일정 개수 이상은 휴리스틱으로 전환합니다.
화면 수 증가챗봇은 부가 기능으로 유지하고 핵심 화면 안정화 이후 착수합니다.

4주 로드맵.

지도 연동과 데이터 수집부터 동선 계산, 마이페이지, 발표 준비까지 단계적으로 완성합니다.

1주차

백엔드 API/DB 기본 구조 설계, 회원 테이블 포함, 네이버 지도 연동, 데이터 수집 착수.

2주차

로그인/회원가입 구현, 빵집 선택 → 상위 3개 동선 계산 로직 구현, 결과 화면 완성.

3주차

도보·차량·버스별 근사 소요시간 구현, 마이페이지 개발, UI 완성도 개선.

4주차

버그 수정, 실제 API 교체 시도, 챗봇 부가 기능 연결, 데모와 발표 준비.
