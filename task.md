# 빵지도 — Task 백로그

1주차(기획/프로토타입)를 마치고 2주차부터 본격 개발에 들어가기 전, 무엇을 언제 할지 미리 정리한 문서입니다.

## 1. 백로그(Backlog)란?

**정의**: 아직 완료하지 않은, 앞으로 해야 할 작업들을 우선순위 순으로 쌓아둔 목록. 애자일/스크럼에서 쓰는 개념으로, "제품 백로그(전체 기능 목록)"와 "스프린트 백로그(이번 주기에 할 일만 뽑은 목록)"로 나뉘는 경우가 많습니다.

**왜 필요한가**: 개발 중에는 항상 "이것도 해야 하지 않나?" 하는 생각이 계속 떠오릅니다. 백로그가 없으면 이런 생각이 들 때마다 하던 작업을 멈추고 새 작업으로 옮겨가게 되어 진행 중인 것도 다 못 끝내고 계획도 흔들립니다. 백로그에 일단 적어두고 우선순위대로 순서를 지키면, 지금 할 일이 아니라는 걸 확인하고 하던 일에 계속 집중할 수 있습니다.

**어떻게 생겼나**: 보통 아래 정보를 가진 카드/행의 목록입니다.
- 작업 이름 (한 문장으로 무엇을 하는지)
- 우선순위 (지금 꼭 해야 하는지, 나중에 해도 되는지)
- 상태 (할 일 / 진행 중 / 완료)
- 예상 소요 기간 또는 주차

**어떻게 관리하나**: Notion, Google Sheet, Jira/Trello 같은 칸반보드, 혹은 GitHub Issues/Projects 등 다양한 도구를 씁니다. 핵심은 도구가 아니라 "모든 할 일이 한 군데 모여 있고, 우선순위가 보이고, 진행 상태를 업데이트한다"는 것입니다.

## 2. 이 프로젝트의 관리 방식

- 이 파일(`docs/빵지도_Task_백로그.md`)로 관리하며, 코드와 같은 저장소에서 버전 관리됩니다. README에 아래처럼 링크를 겁니다.
  ```md
  ## Task 백로그
  개발 일정과 작업 목록은 [docs/빵지도_Task_백로그.md](./docs/빵지도_Task_백로그.md) 에서 관리합니다.
  ```
- 작업 상태는 체크박스(`- [ ]` / `- [x]`)로 표시하고, 매주 실제로 끝난 것에 체크하며 다음 주 계획을 조정합니다.
- 팀/협업 규모가 커지면 GitHub Issues나 Notion 보드로 옮기는 것도 고려할 수 있지만, 지금 규모(개인 프로젝트, 4주)에서는 markdown 하나로 충분합니다.
